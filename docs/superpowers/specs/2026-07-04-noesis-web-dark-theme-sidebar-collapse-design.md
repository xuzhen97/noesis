# Noesis Web 暗色主题可读性与侧栏收起设计

## 背景

当前 Noesis Web 控制台已经具备 Owner Token 门禁、控制台外壳、暗色/亮色主题、仪表盘和最小导航。用户在查看暗色主题截图后反馈：亮色主题没有明显问题，但暗色主题不够易读，有“闪眼”的感觉；同时希望在左下角增加一个按钮，用于收起左侧菜单。

本次设计聚焦两个小而明确的体验优化：降低暗色主题 glare，并为桌面侧栏增加可收起状态。它不改变 Noesis 的领域语义、登录方式、路由结构或页面信息架构。

## 已确认目标

### 包含

- 优化暗色主题可读性。
- 保留 Noesis 的科技控制台气质，但从高饱和科技屏降为更耐看的生产工具暗色。
- 桌面端左侧菜单支持展开/收起。
- 收起后变成图标窄栏，导航入口不丢失。
- 收起状态保存在当前浏览器本地。
- 继续支持亮色主题，不主动重做亮色视觉。

### 不包含

- 不做移动端抽屉菜单。
- 不引入 tooltip 组件；收起态先用 `title` 和 `aria-label` 保持可发现性与可访问性。
- 不重做仪表盘信息架构。
- 不新增 mock 数据。
- 不改变 Owner Token 登录门禁语义。
- 不接入真实 Gateway API。

## 设计原则

本次沿用 `ui-ux-pro-max` 的优先级：可访问性、触控与交互、布局响应式、暗色配色可读性优先于装饰效果。

- 暗色主题的功能目标是长时间查看，而不是展示页视觉冲击。
- 科技感可以来自结构、细线、层次和克制的环境光，不依赖高亮 glow。
- 导航收起应节省空间，但不能牺牲主要入口可达性。
- 新交互必须能键盘操作，并具备清晰的 `aria-label`。

## 暗色主题调整

暗色主题方向从“高饱和蓝紫/青色 glow”调整为“生产工具暗色 + 克制科技感”。

### 背景

- 保留深色底色。
- 保留蓝紫和青色环境光，但降低透明度和覆盖范围。
- 网格纹理降为非常淡的背景层，避免在主内容区域造成视觉噪音。
- 避免大面积高饱和紫色铺在左侧导航和主内容背后。

### 面板与阴影

- 卡片仍使用半透明深色面板、细描边和轻微层次。
- 降低 `shadow-2xl` 和强 glow 的存在感。
- hover 状态只做轻微抬升或边框变化，不产生明显发光。

### 导航 active 状态

- 保留左边线与淡背景，作为当前页面的明确提示。
- 减弱紫色外发光，避免侧栏左侧成为视觉噪点。
- 收起态只高亮当前图标所在项，不扩大 glow 区域。

### 文字可读性

- 提高正文和次级文字在暗色背景上的可读性。
- 避免灰紫色文字过暗或过淡。
- 继续通过主题 token 管理颜色，不在组件里散落硬编码色值。

### 亮色主题

亮色主题不作为本次重点。改动不得破坏亮色主题现有可用性；如果暗色 token 需要专门调整，应尽量限制在 `.dark` 分支或暗色相关 utility 中。

## 侧栏收起交互

桌面端侧栏支持两种状态：展开和收起。

### 展开态

- 宽度保持当前约 `240px`。
- 顶部显示 Noesis 图标、`Noesis 灵识` 和副标题。
- 导航项显示图标和文字。
- 底部显示“收起菜单”按钮。

### 收起态

- 宽度约 `72px`。
- 顶部只保留 Noesis 图标。
- 导航项只显示图标，隐藏文字。
- 每个导航项保留 `title`，提供浏览器原生 tooltip。
- 底部按钮变为“展开菜单”。
- 主内容区域随 grid 宽度变化获得更多空间。

### 收起按钮

- 位置：桌面侧栏底部区域，视觉上接近左下角。
- 图标：使用 lucide 的 `PanelLeftClose` 和 `PanelLeftOpen`。
- 文案：展开态显示“收起菜单”，收起态可只显示图标。
- 可访问性：按钮必须有准确 `aria-label`。
- 交互尺寸：不小于 44px。

### 动效

- 宽度、间距和可见内容变化使用约 200ms 过渡。
- 尊重 `prefers-reduced-motion`；用户减少动效时不做明显过渡。
- 不使用会造成布局抖动的复杂动画。

## 本地状态

新增一个本地 UI 偏好：

- key：`localStorage["noesis.sidebarCollapsed"]`
- value：字符串 `"true"` / `"false"`
- 默认值：没有存储值时默认展开，即 `false`

读写规则：

- localStorage 可用时，用户切换侧栏状态后保存。
- localStorage 不可用时，按钮仍可改变当前 React state；刷新后不保证记忆。
- 非 `"true"` 的值都按 `false` 处理，避免异常输入导致界面不可用。

建议在现有 `session.ts` 中按已有 helper 风格新增：

- `readSidebarCollapsed(storage)`
- `saveSidebarCollapsed(storage, collapsed)`

并在 `session.test.ts` 增加最小覆盖：默认 false、保存 true/false、存储异常 fallback。

## 响应式规则

- 侧栏收起按钮只在 `lg` 以上桌面侧栏显示。
- 移动端继续使用当前顶部横向导航，不显示侧栏收起按钮。
- 375px 小屏不得出现横向滚动。
- 收起态只影响桌面 grid 布局，不改变移动端导航策略。

## 技术边界

本次是对现有控制台壳的增量优化：

- 主要修改 `packages/web/src/styles.css` 和 `packages/web/src/App.tsx`。
- 可按需要扩展 `packages/web/src/session.ts` 和 `packages/web/src/session.test.ts`。
- 不新增页面目录、数据层、API 层或复杂导航系统。
- 不新增第三方依赖。

## 验证标准

自动验证：

- `pnpm --filter @noesis/web test` 通过。
- `pnpm verify` 通过。

浏览器验证：

- 暗色 dashboard 可读，背景光晕和网格不再刺眼。
- 亮色主题仍可切换并可读。
- 桌面侧栏可展开/收起。
- 刷新后保持上次侧栏状态。
- 收起后导航仍可进入仪表盘、机器、任务、设置。
- 收起按钮具备准确 `aria-label`。
- 375px 小屏无横向滚动，且不显示桌面收起按钮。

## 决策记录

- 选择“生产工具暗色”而非继续强化赛博视觉，因为用户明确反馈暗色主题闪眼，当前阶段更需要可读和耐看。
- 选择“图标窄栏”而非完全隐藏或抽屉，因为它节省空间但保留导航可达性，符合控制台产品习惯。
- 选择本地持久化侧栏状态，因为它是 UI 偏好，和现有主题偏好一致。
